To get the best developer experience when working with the Nuxt Shopify module, set up your IDE to understand the
GraphQL schemas provided by Shopify. You then get autocompletion, validation, and hover documentation directly inside
your GraphQL operations.

## VSCode Setup

Install the [GraphQL: Language Feature Support](https://marketplace.visualstudio.com/items?itemName=GraphQL.vscode-graphql)
extension for autocompletion, validation and go-to-definition.

For syntax highlighting of GraphQL inside template literals and standalone documents, also install the
[GraphQL: Syntax Highlighting](https://marketplace.visualstudio.com/items?itemName=GraphQL.vscode-graphql-syntax)
extension.

## GraphQL Config

The module generates the GraphQL config for you. On every `nuxt prepare`, `nuxt dev` and `nuxt build` it writes a
`.nuxt/graphql.config.mjs` file containing one [graphql-config](https://the-guild.dev/graphql/config) project per
configured client. Each project wires that client's generated schema (`.nuxt/schema/<client>.schema.json`) to the
files where you write its operations, so the language server routes every file to the correct API.

Because editors read their GraphQL config from the **workspace root**, add a one-line file there that re-exports the
generated config:

```js [graphql.config.mjs]
export { default } from './.nuxt/graphql.config.mjs'
```

::note
The [starter template](/getting-started/installation) already includes this file, so a project created from it works with zero setup. After adding the file for the first time, restart the language server (Command Palette → **VSCode GraphQL: Manual Restart** or your editor's equivalent).
::

If you'd rather manage your own GraphQL config, disable generation:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  shopify: {
    graphql: {
      generateConfig: false,
    },
  },
})
```

## Writing operations

The language server picks up GraphQL operations written as a `#graphql`-prefixed template literal, a template literal preceded by a `/* GraphQL */` comment, or a `gql`/`graphql` tagged template:

```ts
export const PRODUCT_FRAGMENT = `#graphql
  fragment ProductFields on Product {
    id
    title
  }
`
```

```ts
export const PRODUCT_FRAGMENT = /* GraphQL */ `
  fragment ProductFields on Product {
    id
    title
  }
`
```
